NIST-SIP 1.2 -- SIP Libraries and Tools for the People!

What's in this Distribution?

This distribution contains JAVA(TM) libraries for the following:
  • The official Reference Implementation of the JAIN SIP API (JSR 32) maintenance release.
  • The Technology Compatibility Kit (TCK) for the maintenance release of JSR 32.
  • An implementation of the JAIN SDP API (JSR 141).
The Session Initiation Protocol (RFC 3261) is a text based signaling protocol that is widely used for IP Telephony, Instant Messaging and and Internet Conferencing applications.

The Session Description Protocol ( RFC 2327 ) is used for describing media sessions and is frequently used in conjunction with SIP.

Primarily, this distribution contains a protocol stack/library that will help you build and test SIP applications and servers.

Core Library and Stack Packages

This  packages included in the gov.nist.javax.sip hierarchy form the reference implementation for JAIN-SIP 1.1 JSR-32 maintenance release. JAIN-SIP is a low level protocol API for SIP.

Technology Compatibility Kit

The source code for the technology compatibility kit is provided in the directory test/tck.

Examples

These include the following pure JAIN applications:
  • A public domain proxy and registrar with presence and Instant messaging support.
  • A public domain Instant Messaging user agent which inter-operates with Microsoft Messenger
  • A video Soft phone that uses the Java Media Framework

Tools

A trace viewer application that can be used to view the traces generated by the NIST Stack This is our primary debugging tool. To report stack problems please send us a debug log which can be input to the trace viewer to determine the problem.

How to build it

Supported Platforms

The specification states that JAIN-SIP-1.1 needs J2SE 1.4.x If your platform will support it, download and install J2SDK 1.4.1 or above. It is highly recommended that you 1.4.x use it as it is much faster. However, you can also build NIST-SIP under JDK 1.3.1 (for use on a Compaq IPAQ class machine, for example). NIST-SIP-1.2 compiles and runs on the Ipaq using the Blackdown jvm (1.3 compatible) running on top of Familiar (Linux embedded operating system). To add Swing support, you just need to install an optional library.

Building it from Scratch

Get the source from CVS or the nightly snapshot. CVS is recommended. Visit the stable repository on java.net and follow the instructions there to get the source code. The preferred build tool is GNU make but you can also use Jakarta Ant. The ant build.xml file may be out of date.

If you are using Windows (XP, 2000 or NT) please install Cygwin from Redhat (http://www.cygwin.com). and make sure that you include the build package and bash when you install it). The build tool of choice is GNU make and it is executed from a cygwin bash shell.

  • First, get and install J2SE (click on the java logo on the right).

    • Use the makefile provided to build everything:
    • edit build-config - adjust the paths to libraries and J2SDK and the xerces and junit libraries. Get the xerces and junit libraries and install them in the directories libs/junit and libs/xerces respectively. These are available from the junit.org website and apache websites. Else, you can download a (possibly out of date) copy of these from NIST and unzip them at the root of the distribution tree (which will create a libs directory for you).
    • The following make targets are relevant:
      • make ri makes just the reference implementation for JAIN-SIP. A jar file nist-sip-1.2.jar will be generated which is the RI for JAIN-SIP 1.1.
      • make javadoc makes the documentation
      • make tools makes the tools
      • make examples makes the examples.
      • make classfiles makes just the .classes (no docs).
  • To build the TCK, please follow the instructions above and generate the classes for the RI (ie. use the classfiles target). You need to get and install the junit library in lib/junit to build the TCK. It may be downloaded from the junit.org web site . Here is how you can build the TCK:
    • make the RI using the instructions above.
    • make the TCK using the target "tck". All the classes needed for the tck are included in jain-sip-1.1.tck.jar.
    • You can run the TCK in text or GUI (interactive) mode. In GUI mode, run the tck against the test implementation using the following command (assuming the class path includes the implementation under test):
      java -jar jain-sip-1.1.tck.jar -mode gui
      A user interface appears and allows you to enter the implementation path.
      Alternatively, in text mode:
      java -jar jain-sip-1.1.tck.jar -mode text [-path impl_path]
      Where impl_path is the path to your implementation (eg. gov.nist)
    • java -jar jain-sip-1.1.tck.jar -help for more details on command line options.

Reporting Bugs

To report bugs you need to enable debugging and message logging. Set the gov.nist.javax.sip.TRACE_LEVEL property to 32 to enable debugging (see documentation for gov.nist.javax.SipStackImpl.java). Set the gov.nist.javax.sip.DEBUG_LOG property to a writeable file name (eg. debuglog.txt). Send us the generated debug log file to help us determine the problem. Post an issue to the bug tracker at the jain-sip project at java.net so we can better track bugs, and email us at nist-sip-dev@antd.nist.gov to notify us. Note that you need to join the project on java.net as an observer to post an issue. Follow the instructions on the java.net website above to do this.

Release Status

JSR 32 is in the "maintenance" phase although essentially, this is a complete re-write of the spec.

JAIN-SIP 1.1 is now declared "final". This means that the API is frozen and the RI and TCK meet the approval of the JCP PMO. There are numerous small errors in documentation of the JAIN-SIP Spec. See here for an unofficial list of documentation errata.
